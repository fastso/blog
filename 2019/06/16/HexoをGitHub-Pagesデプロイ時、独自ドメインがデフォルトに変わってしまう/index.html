HexoをGitHub Pagesデプロイ時、独自ドメインがデフォルトに変わってしまう

GitHub Pagesに独自ドメインを設定

GitHub Pagesに独自ドメインを設定する手順は、ほかのサイトで紹介されているのでここでは述べない。
以下のサイトをご参考まで。

  • GitHub Pages + 独自ドメイン + HTTPS (SSL) のサポートをGitHubが提供開始
  • Setting up an apex domain (GitHub公式)

独自ドメイン設定後に、Hexoを使用してblogを開設したが、
デプロイするたびにGitHubの独自ドメイン設定が消えて、デフォルトドメインに変わってしまう現象があった。

独自ドメイン設定が消えないために

独自ドメインを設定した後、GitHubリポジトリにCNAMEというファイルが作らる。
CNAMEファイルの中に独自ドメインが保管されているのを確認した。

CNAMEファイルはローカルにあるHexoのpublicフォルダになく、だからデプロイ時に消えてしまうのが原因だとわかった。

そのため以下の対策をした。

  • GitHubリポジトリのCNAMEファイルをダウンロード
  • CNAMEファイルをローカルのpublicフォルダに配置

上記で問題解決できた。
