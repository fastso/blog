元程序设计 (Metaprogramming)

没有多少天才可以克服对细节的关注。 –Levy第八法则

No amount of genius can overcome a preoccupation with detail. –Levy’s Eighth Law

动态配置 (Dynamic Configuration)

要用元数据描述应用的配置选项：调谐参数，用户偏好，安装目录等等。
Use metadata to describe configuration options for an application: tuning parameters, user preferences, the installation directory, and so on.

Metadata是什么？
严格的说，Metadata是关于数据的数据。
从宽泛的意义上来讲，Metadata是任何对应用进行描述的数据–应用该怎样运行，它应该使用什么资源等等。典型的情况下，Metadata在运行时，而不是编译时被访问和使用。

元数据驱动的应用 (Metadata-Driven Applications)

元数据驱动的好处：

  • 迫使解耦，带来更灵活，可适应性更好的程序。
  • 迫使推迟细节处理，创建更强健，更抽象的设计。
  • 无需重新编译应用。
  • 可以通过接近问题领域 (problem domain) 的方式表示元数据。
  • 可以使用相同的应用引擎和不同的元数据，实现不同的项目。
