Interval DP (区间DP)

概要

在一段区间上的dp，通过枚举左右子区间来求解。

方法

先求出小区间（部分）最优解，然后逐步加大区间的范围，将小区间合并成稍微大点的大区间，最后合成总区间。
dp[l][r]储存区间[l,r]的状态。

复杂度一般为O(n^3)。(某些可以优化为O(n^2))

Oracle Cloud Applications ロール

ロールベースのアクセス制御 ( Role-Based Access Control(RBAC) )

Oracle Cloud Applicationsは、ロールベースのアクセス制御(RBAC)を使用します。

ロールの種類

Oracle Cloud Applicationsでは、5種類のロールが定義されています。

種類 Type 説明 例 ほかのロールを継承 ユーザに付与 事前定義 新規作成・編集 備考
データロール(HCM用) Data roles 労働者のジョブと労働者がアクセスする必要があるデータとの結合 HCM dataロール「Country Human Resource Specialist」は、ジョブ（人事スペシャリスト）とデータスコープ（国）の組合せ X X ✔ ✔
抽象ロール Abstract roles 会社階層内の労働者の階級に対応 従業員、ラインマネージャー、派遣社員 ✔ ✔ ✔ ✔ すべての労働者は少なくとも一つを所有
ジョブ・ロール Job roles 労働者が行う特定のジョブに対応 人事アナリスト、給与管理者 ✔ ✔ ✔ ✔
集計権限 Aggregate privileges 個別のタスクまたは職務に対する1つの機能権限と関連するデータセキュリティポリシーとの組合せ。集計権限が提供する機能権限は、タスクフロー、アプリケーション画面、作業領域、レポート、バッチプログラムなどへのアクセスを許可できます。 X X ✔ X

職務ロールについて

機能セキュリティ権限とデータセキュリティポリシーは、職務ロールによって付与されます。

機能セキュリティ権限

機能を使用するための権限のこと。

一部のユーザインタフェースはデータセキュリティの対象ではないため、一部の機能セキュリティ権限には同等のデータセキュリティポリシーがありません。

データセキュリティポリシー

  • ロール。データセキュリティポリシーは、ジョブ・ロール、抽象ロール、職務ロールと集計権限に付与できます。
  • アクセスされているビジネスオブジェクト。データセキュリティポリシーは、リソースをテーブル名で識別します。
  • ビジネスオブジェクトの特定インスタンスへのアクセスを制御する条件。
  • データセキュリティ権限は、データに対してのアクション許可を定義する。

事前定義の職務ロール

事前定義の職務ロールは、グループして管理したいかもの権限グループ。

例：事前定義の職務ロール「親」に、職務ロール「子」が含まれ、「子」の権限を外したい場合以下の手順を実施します。

  1. 「親」をコピーします。
  2. コピーした職務ロールから「子」を削除します。

集計権限について

集計権限の名称

集計権限の名称は、含まれている機能セキュリティ権限の名称に基づく。

例：「Promote Worker」集計権限には、「Promote Worker」機能セキュリティ権限が含まれています。

ロール階層への適用

ジョブ・ロールと抽象ロールのほとんどの機能とデータセキュリティは、集計権限にて提供され、かつ集計権限はほかのロールを継承できないため、ロール階層は少なく済みます。

カスタムロールへの適用

集計権限をビルドインブロックとして使用できます。

新規作成、編集、コピー

集計権限は新規作成、編集、コピーできません。
集計権限を単一エンティティとして使用する必要があります。

ジョブ・ロールまたは抽象ロールをコピーする場合、コピー元ロールの集計権限はコピーされません。 その代わり、コピーされたロールの集計権限にロールメンバーシップが自動追加されます。

セキュリティ構成の考慮事項

Oracle Cloud Applications製品には、ジョブ・ロールと抽象ロールがデジタル・キーリングとして機能することで、セキュリティを強化します。
各製品は、単独のセキュリティリファレンスがあります。

また、全ての製品に共通のロールがあります。

  • Application Implementation Consultant
  • IT Security Manager

ロールはロールコードの接頭辞で簡単に識別できます。（接頭辞の例：ORA_）

所属企業のジョブに適合しない場合

カスタムジョブ・ロールを新規作成します。適切な集計権限と職務ロールを追加してください。

新しい権限が必要な場合

独自バージョンを作成します。事前定義のロールをコピーし、編集します。必要に応じて集計権限、職務ロール、機能セキュリティ権限、およびデータセキュリティポリシーを追加または削除できます。

Warshall-Floyd algorithm (华沙弗洛伊德算法)

概要

Warshall-Floyd算法是解决任意两点间的最短路径（所有顶点最短路径）算法，可以在任何图中使用，包括有向图、带负权边的图。

方法

  • Input
    • 有向或者无向图 G = (V, E)
    • E的各边的长度
  • Output
    • 输出连接顶点i和顶点j的最短路径（i,j属于V）

使用dp解决此问题。
首先进行初始化dp[i][j]为顶点i到顶点j的距离，如果不存在这条边则设为∞。
从第一个顶点开始，依次将每个顶点作为媒介k，然后对于每一对顶点i和j，
查看其是否存在一条经过k的，距离比已知路径更短的路径，如果存在则更新它。

复杂度为O(V**3)。

Dijkstra's algorithm (狄克斯特拉算法)

概要

狄克斯特拉算法是从一个顶点到其余各顶点的最短路径算法，用于不包含负边的有权图中的单源最短路径问题。
类似于BFS（宽度优先搜索）。

方法

  • 准备priority_queue(优先级队列)，push起点和到其的最短距离的信息（起点为00, 其他顶点为∞）。因为使用了优先级队列，所以信息以最短距离顺序排序。
  • 重复以下的操作，直到优先级队列为空集。
    1. 提取优先级队列的先头元素（距离最短的元素）
    2. 如果所提取的元素中包含的距离信息大于所记录的最短距离，则该操作终止，并且操作返回到提取前导元素的操作。
    3. 扫描可以从那里移动的顶点，如果有可以更新最短距离的顶点，则更新该距离，然后将把走到该顶点的距离，顶点的信息push到优先队列。

复杂度为O(ElogV)。

Pythonic Thinking

Item 1: Know Which Version of Python You’re Using

  • major versions: Python 2 and Python 3
  • popular runtimes: CPython, Jython, IronPython, PyPy, etc.

Item 2: Follow the PEP 8 Style Guide

PEP 8 Style Guide

Item 3: Know the Differences Between bytes, str, and unicode

  • bytes contains sequences of 8-bit values
  • str contains sequences of Unicode characters

  • bytes and str instances can’t be used together with operators (like > or +).

  • If you want to read or write binary data to/from a file, always open the file using a binary mode (like ‘rb’ or ‘wb’).

Item 4: Write Helper Functions Instead of Complex Expressions

  • Move complex expressions into helper functions, especially if you need to use the same logic repeatedly.

Item 5: Know How to Slice Sequences

  • Assigning to a list slice will replace that range in the original sequence with what’s referenced even if their lengths are different.

Item 6: Avoid Using start, end, and stride in a Single Slice

  • Prefer using positive stride values in slices without start or end indexes. Avoid negative stride values if possible.

Item 7: Use List Comprehensions Instead of map and filter

  • List comprehensions are clearer than the map and filter built-in functions because they don’t require extra lambda expressions.
  • List comprehensions allow you to easily skip items from the input list, a behavior map doesn’t support without help from filter.

Item 8: Avoid More Than Two Expressions in List Comprehensions

  • List comprehensions support multiple levels of loops and multiple conditions per loop level.
  • List comprehensions with more than two expressions are very difficult to read and should be avoided.

Item 9: Consider Generator Expressions for Large Comprehensions

  • List comprehensions can cause problems for large inputs by using too much memory.
  • Generator expressions avoid memory issues by producing outputs one at a time as an iterator.

Item 10: Prefer enumerate Over range

Prefer to use enumerate to iterate both index and value of the list.

Item 11: Use zip to Process Iterators in Parallel

  • The zip built-in function can be used to iterate over multiple iterators in parallel.
  • In Python 3, zip is a lazy generator that produces tuples. In Python 2, zip returns the full result as a list of tuples.
  • zip truncates its output silently if you supply it with iterators of different lengths.
  • The zip_longest function from the itertools built-in module lets you iterate over multiple iterators in parallel regardless of their lengths.

Item 12: Avoid else Blocks After for and while Loops

  • Python has special syntax that allows else blocks to immediately follow for and while loop interior blocks.
  • The else block after a loop only runs if the loop body did not encounter a break statement.
  • Avoid using else blocks after loops because their behavior isn’t intuitive and can be confusing.

Item 13: Take Advantage of Each Block in try/except/else/finally

  • The try/finally compound statement lets you run cleanup code regardless of whether exceptions were raised in the try block.
  • The else block helps you minimize the amount of code in try blocks and visually distinguish the success case from the try/except blocks.
  • An else block can be used to perform additional actions after a successful try block but before common cleanup in a finally block.

Feature Preprocessing and Generation with respect to models

Feature preprocessing and generation with respect to models

  • Feature Preprocessing
  • New Feature Generation

Both feature preprocessing and feature generation depend on the model you are going to use.

  • numeric features (数字)
  • categorical features (分類)
  • datatime features (日時)
  • coordinate features (座標)

Numric features

Preprocessing

  • Scaling and Rank for numeric features:

    • Tree-based models doesn’t depend on them.
    • Non-tree-based models hugely depend on them.

  • MinMaxScaler - to [0,1]

    • X = (X X.min())/(X.max() X.min())
    • sklearn.preprocessing.MinMaxScaler
  • StandardScaler - to mean==0, std==1
    • X = (X X.mean())/X.std()
    • sklearn.preprocessing.StandardScaler
  • Rank - sets spaces between sorted values to be equal
  • np.log(1+x) and np.sqrt(1+x)

Feature generation

  • Prior knowledge
  • EDA (Exploratory Data Analysis)

Categorical features

  • Values in ordinal features are sorted in some meaningful order.
  • Label encoding maps categories to numbers.
  • Frequency encoding maps categories to their frequencies.
  • Label and Frequency encodings are often used for treebased models.
  • One-hot encoding is often used for non-tree-based models.
  • Interactions of categorical features can help linear models and KNN.

Datatime and Coordinate features

  • Datetime
    • Periodicity
    • Time since row-independent/dependent event
    • Difference between dates
  • Coordinates
    • Interesting places from train/test data or additional data
    • Centers of clusters
    • Aggregated statistics

Handling missing values

  • The choice of method to fill NaN depends on the situation
  • Usual way to deal with missing values is to replace them with -999, mean or median
  • Missing values already can be replaced with something by organizers
  • Binary feature “isnull” can be beneficial
  • In general, avoid filling nans before feature generation
  • Xgboost can handle NaN

Feature extraction from text and images

Texts

  • Preprocessing
    • Lowercase, stemming, lemmarization, stopwords
  • Bag of words
    • Huge vectors
    • Ngrams can help to use local context
    • TFiDF can be of use as postprocessing
  • Word2vec
    • Relatively small vectors
    • Pretrained models

Images

  • Features can be extracted from different layers
  • Careful choosing of pretrained network can help
  • Finetuning allows to refine pretrained models
  • Data augmentation can improve the model

代表的な財務諸表

三大財務諸表

貸借対照表 (B/S : Balance Sheet)

決算日など”一時点”における資産・負債・純資産の財政状態を表す。
「今何を保有しているか」を示すシート。

  1. 流動比率（%）＝（流動資産 ÷ 流動負債） × 100
    企業の短期の支払能力を表す指標。200%以上が望ましい。
  2. 当座比率（%）＝（当座資産 ÷ 流動負債） × 100
    企業の短期の支払能力を表す指標。100%以上が望ましい。
  3. 固定比率（%）＝（固定資産 ÷ 自己資本） × 100
    長期にわたって回収が必要な固定資産投資と自己資本の割合を示すもので、財務安全性を表す指標。100%以下が望ましい。
  4. 固定長期適合率（%） ＝（固定資産 ÷（自己資本 ＋ 固定負債）） × 100
    財務安全性を表す指標。100%以下でなければ厳しい。
  5. 負債比率（%）＝（負債 ÷ 自己資本） × 100
    財務安全性を表す指標。100%以下が望ましい。
  6. 自己資本比率（%）＝（自己資本 ÷ 資産） × 100
    自己資本と借入金などの負債合計（他人資本）との比率。50%以上が望ましい。

損益計算書 (P/L : Profit and Loss Statement)

”一定期間”の営業成績を表す。
最大の特徴は”５つの利益”が示されている。

  1. 売上総利益 = 売上高 - 売上原価
    ”粗利”のこと。
  2. 営業利益 = 売上総利益 - 販売費及び一般管理費
    本業で得られた利益。マイナスの場合、過渡期のものなら問題ないが、そうでないならビジネスモデルが破綻している可能性がある。
  3. 経常利益 = 営業利益 + (営業外収益 + 営業外費用)
    本業に投資活動の損益を加味した利益。
  4. 税引前当期純利益 = 経常利益 + (特別利益 - 特別損失)
    本業、投資活動に加え、単年度の突発的な損益を加味した利益。
  5. 当期純利益 = 税引前当期純利益 - 法人税等
    税金を支払った後に残った利益。

簿記

簿記一巡の手続き

  • 仕訳
    日々発生する取引の発生順に、適切な”勘定科目”を用いて借方要素と貸方要素に分けて記録していく。
  • 転記
    仕訳帳は発生順なので、これを勘定ごとに集計し、総勘定元帳を作成する。
  • 試算表の作成
    勘定ごとに合計し、その残高を求めて合計残高試算表を作成する。
  • 決算
    決算整理仕訳を行う。
  • 勘定の締切り
    決算整理仕訳を行った後、各勘定を締め切る。その後、繰越試算表を作成して時期に備える。
  • 財務諸表の作成
    貸借対照表と損益計算書、そのほか必要に応じて財務諸表を作成する。

決算

会社は通常１年間を一つの会計期間として、設立時に定めた事業年度末（期末）に決算を行う。

  • 期首：会計期間の最初の日
  • 期末/決算日：会計期間の最後の日

決算手続き

  1. 各部門での入力締切（仕訳帳の締切）
  2. 試算表の作成
  3. 決算整理事項の調査
  4. 精算表の作成
  5. 決算整理仕訳の帳簿記入
  6. 帳簿締切りの手続き
  7. 財務諸表の作成

決算の確定

  1. 監査
  2. 取締役会（決算取締役会）の承認
  3. 株主総会での承認

連携決算

  1. 子会社から単独決算書、その他の必要情報の収集
  2. 連結調整処理
  3. 資本連結
  4. 相殺処理
  5. 関連会社（持分法適用会社）の処理
  6. 財務諸表作成
  7. セグメント情報の作成
