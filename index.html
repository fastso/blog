职业生涯管理

[GOAL] 本课程学习如何借助了解自己、在职业挫折中寻找机遇，以及养成良好的学习习惯等方式，为自己打造一条满意的职业道路。

1. 你的职业道路

在个人职业生涯这个问题上，你才是最终决定者。

主宰自己的职业生涯

The Art of Career Development 职业发展的艺术

  1. Build your career on a foundation of expertise
  2. Prove the work you do matters
  3. Be courageous and put yourself out there
  4. Connect to a community that teaches, inspires, and supports you

采取“网格式”职业发展方式

如今，职业网格取代了职业阶梯。
在职业网格的发展模式中，员工没有固定的职业发展道路，而是拥有多种可能的发展道路。

职业生涯管理大于找工作

在人的一生中，可能会在多个不同的时间点积极寻找新工作；但职业生涯管理却是一个持续的过程。

我们生活在个性化服务时代。从营销信息到早上的咖啡，一切都根据我们的口味和兴趣进行了个性化定制。

所以，我们对成功的定义应该也要展现个性化。

职业生涯并非脱离你的个人生活而孤立存在的。虽然有些人会狭隘地认为成功就等于赚很多钱，但仍有很多人拥有更开阔的视野。

2. 了解自己

你的主要兴趣、优势和价值观是什么？

你的职业自我

很少人从毕业到退休一直走一条直线式的职业发展路径。

  • 根据美国劳工统计局的数据，当今的普通工作者在 40 岁之前会从事 10 种不同的工作。
  • 大多数工作者的职业与他们在学校主修的领域完全不相关。
  • 现在的许多工作明天可能就会消失，而明天的许多工作现在可能还不存在。

没有人可以笃定地假设，他们现在拥有的技能和专长未来还能用得上。这意味着什么呢？

  • 考虑将培养新技能作为其中一项工作职责
  • 安排时间定期审视自己
  • 别让你的角色来定义你

若你想主宰自己的职业发展路径并决定自己的职业发展重点，可以从以下几个方面展开思考：

  • 主要兴趣
    可以激发你的热情、激励你不断向前的活动
  • 技能
    已经拥有的技能以及想要培养的技能
  • 价值观
    你的指导原则以及这些原则与工作的契合度
  • 工作场所偏好
    你喜欢的工作环境

颠覆自我: 如果你今天中了彩票，你余生或至少接下来的几年会做什么？

是什么，点燃了你的热情

兴趣会驱动你去做会让自己开心的事，因此最终会让你对工作保持热情。

例如，你可能会热衷于持续关注自己喜欢的运动队的统计数据。或者，你可能喜欢语言和想法，并因此开始撰写个人博客。

职业与最主要兴趣能够契合的人，最有可能获得较好的职业满意度。

你拥有或需要哪些技能

不要让技能不足阻碍你追求目标的步伐，因为可以通过培训和练习来掌握技能。

若想养成学习习惯，你可以：

  • 专注于行业所需的最新技能
  • 与其他学习者互动
  • 立即运用你所学的知识
  • 确定要努力实现的里程碑

对你来说，重要的是什么

决定职业满意度的一大因素是你的工作是否与你的价值观一致。

定期评估你的工作和价值观之间的契合度。

完美胜任工作: 将技能、价值观和兴趣与个人职业对齐。

你适合哪种环境

在考虑自己的工作时，找到适合的工作环境同样也很重要。

随着职业生涯的发展，你对工作环境的偏好也可能发生变化。

3. 寻求职业发展支持

关键不在于你认识谁，而在于谁认识你。

培养人际网络

职业人际网络中的每个人脉在支持你个人职业目标的实现中都发挥着不同的作用。有些人脉可能是一种信息源，让你能够了解最新的行业趋势；而其他人脉则可能是你的盟友，会把你的才华推荐给他们的人脉。

许多人脉关系是自然形成的。例如，你通过一次专业会议与某人建立了人际关系，一位前上司成为了你的导师，或者不属于你工作团队的某位公司同事变成了你的拥护者。

只是，一般情况下你还是需要努力维系已建立的人脉关系。

  • 保证多元化
    囊括视角、专长和背景不同的人员。你需要可帮助你评估新观点的人员。
  • 纳入倾听者
    提供建议的人通常很容易找到，但在你说明问题时会认真倾听的人，往往帮助更大。
  • 培养人际关系
    不要仅在有所求时才联系他人：制定计划一起参加某个会议、一起去喝咖啡，以及参加一些诸如生日聚会之类的活动。

拥有许多通过社交媒体构建的人脉并不一定表示你拥有强大的职业人际网络。而且，群发信息无助于建立个人的职业信誉。

组建个人董事会

个人董事会由六到八人组成。在需要确定职业选择时，你可以咨询董事会成员，向他们寻求指引。当你面临机遇、挑战或重要职业决策时，你可以向董事会成员寻求反馈。

最有价值的成员具备以下特质：

  • 提供不同的观点，并且来自工作中的不同领域
  • 在某些方面拥有的能力比你强
  • 在某些方面掌握的知识比你多

只选择会支持你所有决定的人，无助于你的职业发展。

组建个人董事会不一定要遵循一个正式的流程。不需要召开会议，只要选择合适的人选并与之保持联系就好。如果你选择你会定期与之交流的人，那么这种关系的形成会更加自然。

个人董事会是一个特别宝贵的可用资源。在需要帮助时不要害怕寻求帮助。那些优秀的专家成员为你提供的帮助，可能会共同催生出绝佳的结果。

寻求帮助

要想获得成功，关键在于有效利用人脉关系来获得帮助。

在寻求帮助时，请确保：

  • 阐述目标
    让对方了解对他们的工作或经历你具体欣赏什么，以及为何你相信他们能够帮助到你。
  • 注重互惠互利
    说明你可以为对方提供什么，包括知识、见解或支持等。
  • 描述你设想的关系运作方式
    使自己的预期看起来温和一些，并考虑对方的时间。
  • 留出时间让对方回应
    为对方留出一些时间来考虑你的请求，并计划稍后再与他们联系。

发展你的职业人际网络: 你不必一开始就是人际网络构建方面的专家。

4. 成为敏捷学习者

学习如何使自己一直拥有最新技能。

学习，助力成功

在当今不断发展的职场环境中，持续学习是保持竞争力、持续创造价值的首要条件。

如果不持续努力更新自己的技能并学习新技能，你就可能会落后于人。

以玩一项运动或掌握一种乐器为例。练习、重复、反思和反馈都是取得成功的关键因素。对于掌握新的工作技能来说，也是如此。

你需要确保你没有选错学习内容：

  • 思考对组织较有价值的技能是什么
  • 研究同行业其他公司正在追求的技能是什么
  • 制造机会，在技能不足的领域实现自我提升
  • 尝试新技能时，请其他人为你提供反馈

转变思维方式

有些因素会阻碍我们持续学习，而其中部分因素是深植在我们脑海中的想法。

研究表明，除非你相信自己可以培养和发展新技能，否则世界上没有哪种方法可以带来很大的不同。

要转变为成长型思维:

  • 当自己在回避挑战时，要意识到这一点
    当你害怕让自己难堪或失败时，表明你已经陷入固定型思维。
  • 意识到你有其他选择
    你可以望而却步，也可以重新定义挑战，并把挑战视作学习的机会。
  • 与自己的内心对话
    使用反映成长型思维的话语——例如，不说“因为我的数学一直很差，所以我对学习如何使用数据分析不感兴趣”，而说“学习数据分析有助于提高我的市场预测能力，而且如果有不懂的地方，我也可以请同事帮我。””
  • 特意转变为成长型思维
    迎接挑战，寻求反馈并从挫折中汲取教训。

保持敏捷

敏捷的学习者具备以下能力：

  • 保持灵活性
  • 从错误中成长
  • 成功应对挑战

下面这些做法可以帮助你提高学习敏捷力：

  • 寻求个人兴趣，满足自己的好奇心
    你永远无法知道哪些知识可能与你目前的工作或未来的机遇有关。
  • 构建多元化思维的人际网络
    对于那些会给你提供不同方法应对你所面临挑战的个人，将他们加入到你的人际网络中。
  • 抑制自己的自然防御行为
    抑制立即对他人的反馈做出反应的冲动，花时间仔细考量不同的建议。
  • 从失败与成功中汲取经验教训
    想想是否有更好的解决方法，以及今天有效的方法未来是否还会继续有效。

学习如何学习:

  • 志存高远，即渴望学习新事物
  • 不偏不倚的自我认知
  • 永无止境的好奇心
  • 对先苦后甜的接受意愿

在当前工作中拓展自己的技能

在去外面寻找机会之前，可以思考是否有方式可以在当前的工作中拓展个人技能。

在目前的职位上，你是否有机会增加个人价值？你有在不断成长和学习吗？

以下是通过工作塑形来提升学习机会的两种方法：

  • 任务塑形
    改变你的任务量或者改变你不同任务的耗时，以便能花更多时间来完成有较高学习回报的任务。
  • 关系塑形
    改变人际互动模式，以便与观点或技能各异的人一起工作，从而进行学习。

如果你仍然觉得自己的工作没有挑战性，那么可以尝试去组织外部寻找可能的机会。但在此之前，请考虑当前组织内部的其他机会。

5. 扫除职业障碍

将工作中遇到的挫折、瓶颈和不确定性都视为发展机会。

路途崎岖

任何人的职业道路都不可能一帆风顺，路上总是布满荆棘。你可能会遭遇裁员、开除或痛失晋升机会。或者，你可能觉得自己止步不前。

将每个职位视为一个任务，视为职业道路上通往下一个机会的跳板。

如果可以把每个角色视为一种学习和成长机会，你就更有可能获得更高的工作满意度。

  • 投资培养自己的技能
    让自己在当前的工作中获得提升，并为下一份工作做好准备。
  • 发展你的人际网络
    建立持久的人脉关系，因为你永远不知道何时需要用到这些人脉。
  • 保持平衡
    工作并非你的全部，也无法决定你在生活中是否成功。

扭转局面

诚然，在你的工作旅程中，你不可避免地会在某个时刻遭遇挫折或觉得陷入困境。

此时，你可能很想立即做出反应，但请抑制自己的冲动，因为你此时说出的第一句话不管是对你的声誉还是工作来说，都不会有什么好处。

在摆脱了最初的情绪反应之后，你就可以使用以下策略重整自己的思考方式，并将挑战转化为机遇：

  • 转换问题
    将“为什么是我？”等引发自怜的问题转换为行动导向型问题，如“下一步我该怎么做？”或“我能从当前的局面中获得哪些正面益处？”
  • 设定“感受消极时段”
    不要让消极思想消耗自己，而是每天安排特定时段，让自己只能在这个时段才能去想所经历的挫折。而在其他时段中，如果出现消极思想，马上屏蔽，只有等到“感受消极时段”时才允许消极思想的出现。”
  • 尝试从其他视角看问题
    从有利于公司的角度去审视相关事件或决策，让自己在必要时勇于原谅，同时获得有用的见解，让自己避免重复犯相同的错误。

职业空窗期带来的机会: 当你处于职业空窗期时，可以将空窗期视为一个机会，让自己后退一步，并用创造性思维思考自己的职业发展道路。

走出困境

如果你觉得遇到瓶颈——无法学习、无法成长、无法全心投入——并且持续时间超过了几天或几周，那么你可能已经陷入了职业僵局。

以下是克服职业僵局的有效方法：

  • 找出 5 到 7 个能激发你热情的工作
    审视这些工作所代表的共同含义。它们表示：你在寻找新的学习机会吗？还是想要发挥自己在艺术方面的才能呢？使用这些方式来缩短你的列表。
  • 针对缩短后的工作选项收集相关数据
    开展咨询性面谈，研读与你感兴趣的新技能或公司相关的资料。利用你的人际网络，请他们对你所考虑的各种方案提供反馈意见。
  • 寻找机会
    你可以重新设计自己的工作内容，或你目前所在的公司中有更适合你的其它职位吗？如果有，你可以与管理者探讨从事这类职位的可能性。只有在这些方式都不可行之后，才考虑将寻找范围扩大到其他公司。

继续前进

如果你遭遇了挫折或瓶颈，那么要投入时间、下定决心才能让自己继续前进。具体来说，你可以通过以下几个措施找到成功之路：

  • 温习你的电梯演讲
    对让你脱颖而出的闪光点准备一个简洁有力的阐述。
  • 关注你的网络形象
    确保社交媒体上的个人资料包含的都是积极向上的最新信息。
  • 让支持者帮助你建立声誉
    利用有信誉的老同事、导师和其他人脉证明你的实力和人品。

不管你采取哪种方式，都不要诋毁你的前雇主或同事，因为这会让你看起来很不专业。

请记住，挫折是一个绝佳机会，让你可以重新思考哪种工作对你最有意义。一开始，职业轨道的转换可能不会为你带来经济上的回报，但可能会带来更高的工作满意度。

承认自己的作用

你是否曾在获得晋升时感觉惴惴不安？或是收到一封邮件，里面列出了公司发布的一个理想职位，但你却默默将其删除？

如果是的话，你可能和很多人一样患上了所谓的冒名顶替综合症。

冒名顶替综合症: 实际上，觉得自己并不胜任当前的工作，有助于促进你的职业生涯发展。

要克服冒名顶替综合症的感觉，你可以：

  • 专注于你已经取得的成就和想要学习的东西，而非自己的不足。
  • 质疑自己不真实的负面想法，帮助自己克服恐惧。
  • 专注于你为自己的新角色带来的价值。

なぜ、あなたの仕事は終らないのか

人の能力がいきなり向上するようなことはありません。
ならば時間の使い方を徹底的に突き詰めるしかない。
すなわち、時間を制する者が世界を制している。

1. なぜ、あなたの仕事は終らないのか

締め切り間際にラストスパートで仕事を終わらせようとする人の態度を「ラストスパート志向」といいます。のちにお話ししていきますが、ラストスパート志向は、仕事をするうえで最も避けるべきことです。

仕事が終らない人は、得てして後半の応用問題を甘く見ています。

会社を重視しすぎるあまり、仕事の生産効率が落ちている。

「なるはや」をやめれば緊張感が生まれる。

スラック（余裕）がない状態が慢性的に続くと、人はどんどん生産効率が落ちていきます。

仕事が終らない理由

  • 安請け合いしてしまう
  • ギリギリまでやらない
  • 計画の見積をしない

2. 時間を制する者は、世界を制す

どんなに頑張って100%のものを作っても、振り返ればそれは100%でなく90%や80%のものに見えてしまうのです。

プログラマーたちは、100点じゃなくてもいいので90点や80点のプログラムを必ず納期に提出することが求められています。

すべての仕事は、必ずやり直しになる。

石膏像を彫るとき、「眉毛」から始める人はいない。

評価恐怖症にかかった人は、自分の中での100点満点を目指すあまり、本来なら終る仕事も終わらなくなります。

待ち合わせ30分前に、スタバでコーヒーを飲め。締切りの前に締切りがあると考えなければならない。

ビルゲイツが「その問題とこの問題は独立している」とよく言っていたことを覚えています。

どちらの時間の使い方が正しいかをかけた戦いでもある。

時間を制するメリット

  • リスクを測定できる
  • 目に見える形のもの（プロトタイプ）を素早く作ることができる
  • 誤差に対応できる

3. 「ロケットスタート時間術」はこうして生み出された

タイムマシンを作るのに漢字は要らない。

余った時間で、好きなことに打ち込む。

嫌なことをやりたくなければ効率化するしかない。

言葉で説明できなければ、先に形にしてしまえ。

「まず作ってみる」が、未来を変える。

企画を早く形にした者がチャンスをつかめる。

4. 今すぐ実践 ロケットスタート時間術

「ラストスパート志向」が諸悪の根源

多くの人が、「最初はのんびりしていても、最後に頑張ればなんとかなる」という根本的な過ちを改めるところから始めないといけません。

まずは「締切りは絶対に守るもの」と考える

  1. 「まずはどのくらいかかるかやってみるので、スケジュールの割り出しのために2日ください」と答えて仕事に取り掛かる。
  2. その2日をロケットスタート期間として使い、2日で「ほぼ完成」まで持っていく。
  3. 万が一、その2日で「ほぼ完成」まで持っていけなかった場合、これを「危機的な状況」と認識してスケジュールの見直しを交渉する。

スタートダッシュで一気に作る

仕事が終らなくなる原因の9割は、締切り間際の「ラストスパート」が原因です。

考えてから手を動かすのではなく、手を動かしながら考えてください。崖から飛び降りながら飛行機を組み立てるのです。

職務ロールについて

機能セキュリティ権限とデータセキュリティポリシーは、職務ロールによって付与されます。

機能セキュリティ権限

機能を使用するための権限のこと。

一部のユーザインタフェースはデータセキュリティの対象ではないため、一部の機能セキュリティ権限には同等のデータセキュリティポリシーがありません。

データセキュリティポリシー

  • ロール。データセキュリティポリシーは、ジョブ・ロール、抽象ロール、職務ロールと集計権限に付与できます。
  • アクセスされているビジネスオブジェクト。データセキュリティポリシーは、リソースをテーブル名で識別します。
  • ビジネスオブジェクトの特定インスタンスへのアクセスを制御する条件。
  • データセキュリティ権限は、データに対してのアクション許可を定義する。

事前定義の職務ロール

事前定義の職務ロールは、グループして管理したいかもの権限グループ。

例：事前定義の職務ロール「親」に、職務ロール「子」が含まれ、「子」の権限を外したい場合以下の手順を実施します。

  1. 「親」をコピーします。
  2. コピーした職務ロールから「子」を削除します。

集計権限について

集計権限の名称

集計権限の名称は、含まれている機能セキュリティ権限の名称に基づく。

例：「Promote Worker」集計権限には、「Promote Worker」機能セキュリティ権限が含まれています。

ロール階層への適用

ジョブ・ロールと抽象ロールのほとんどの機能とデータセキュリティは、集計権限にて提供され、かつ集計権限はほかのロールを継承できないため、ロール階層は少なく済みます。

カスタムロールへの適用

集計権限をビルドインブロックとして使用できます。

新規作成、編集、コピー

集計権限は新規作成、編集、コピーできません。
集計権限を単一エンティティとして使用する必要があります。

ジョブ・ロールまたは抽象ロールをコピーする場合、コピー元ロールの集計権限はコピーされません。 その代わり、コピーされたロールの集計権限にロールメンバーシップが自動追加されます。

Warshall-Floyd algorithm (华沙弗洛伊德算法)

概要

Warshall-Floyd算法是解决任意两点间的最短路径（所有顶点最短路径）算法，可以在任何图中使用，包括有向图、带负权边的图。

方法

  • Input
    • 有向或者无向图 G = (V, E)
    • E的各边的长度
  • Output
    • 输出连接顶点i和顶点j的最短路径（i,j属于V）

使用dp解决此问题。
首先进行初始化dp[i][j]为顶点i到顶点j的距离，如果不存在这条边则设为∞。
从第一个顶点开始，依次将每个顶点作为媒介k，然后对于每一对顶点i和j，
查看其是否存在一条经过k的，距离比已知路径更短的路径，如果存在则更新它。

复杂度为O(V**3)。

Dijkstra's algorithm (狄克斯特拉算法)

概要

狄克斯特拉算法是从一个顶点到其余各顶点的最短路径算法，用于不包含负边的有权图中的单源最短路径问题。
类似于BFS（宽度优先搜索）。

方法

  • 准备priority_queue(优先级队列)，push起点和到其的最短距离的信息（起点为00, 其他顶点为∞）。因为使用了优先级队列，所以信息以最短距离顺序排序。
  • 重复以下的操作，直到优先级队列为空集。
    1. 提取优先级队列的先头元素（距离最短的元素）
    2. 如果所提取的元素中包含的距离信息大于所记录的最短距离，则该操作终止，并且操作返回到提取前导元素的操作。
    3. 扫描可以从那里移动的顶点，如果有可以更新最短距离的顶点，则更新该距离，然后将把走到该顶点的距离，顶点的信息push到优先队列。

复杂度为O(ElogV)。

Pythonic Thinking

Item 1: Know Which Version of Python You’re Using

  • major versions: Python 2 and Python 3
  • popular runtimes: CPython, Jython, IronPython, PyPy, etc.

Item 2: Follow the PEP 8 Style Guide

PEP 8 Style Guide

Item 3: Know the Differences Between bytes, str, and unicode

  • bytes contains sequences of 8-bit values
  • str contains sequences of Unicode characters

  • bytes and str instances can’t be used together with operators (like > or +).

  • If you want to read or write binary data to/from a file, always open the file using a binary mode (like ‘rb’ or ‘wb’).

Item 4: Write Helper Functions Instead of Complex Expressions

  • Move complex expressions into helper functions, especially if you need to use the same logic repeatedly.

Item 5: Know How to Slice Sequences

  • Assigning to a list slice will replace that range in the original sequence with what’s referenced even if their lengths are different.

Item 6: Avoid Using start, end, and stride in a Single Slice

  • Prefer using positive stride values in slices without start or end indexes. Avoid negative stride values if possible.

Item 7: Use List Comprehensions Instead of map and filter

  • List comprehensions are clearer than the map and filter built-in functions because they don’t require extra lambda expressions.
  • List comprehensions allow you to easily skip items from the input list, a behavior map doesn’t support without help from filter.

Item 8: Avoid More Than Two Expressions in List Comprehensions

  • List comprehensions support multiple levels of loops and multiple conditions per loop level.
  • List comprehensions with more than two expressions are very difficult to read and should be avoided.

Item 9: Consider Generator Expressions for Large Comprehensions

  • List comprehensions can cause problems for large inputs by using too much memory.
  • Generator expressions avoid memory issues by producing outputs one at a time as an iterator.

Item 10: Prefer enumerate Over range

Prefer to use enumerate to iterate both index and value of the list.

Item 11: Use zip to Process Iterators in Parallel

  • The zip built-in function can be used to iterate over multiple iterators in parallel.
  • In Python 3, zip is a lazy generator that produces tuples. In Python 2, zip returns the full result as a list of tuples.
  • zip truncates its output silently if you supply it with iterators of different lengths.
  • The zip_longest function from the itertools built-in module lets you iterate over multiple iterators in parallel regardless of their lengths.

Item 12: Avoid else Blocks After for and while Loops

  • Python has special syntax that allows else blocks to immediately follow for and while loop interior blocks.
  • The else block after a loop only runs if the loop body did not encounter a break statement.
  • Avoid using else blocks after loops because their behavior isn’t intuitive and can be confusing.

Item 13: Take Advantage of Each Block in try/except/else/finally

  • The try/finally compound statement lets you run cleanup code regardless of whether exceptions were raised in the try block.
  • The else block helps you minimize the amount of code in try blocks and visually distinguish the success case from the try/except blocks.
  • An else block can be used to perform additional actions after a successful try block but before common cleanup in a finally block.

Feature Preprocessing and Generation with respect to models

Feature preprocessing and generation with respect to models

  • Feature Preprocessing
  • New Feature Generation

Both feature preprocessing and feature generation depend on the model you are going to use.

  • numeric features (数字)
  • categorical features (分類)
  • datatime features (日時)
  • coordinate features (座標)

Numric features

Preprocessing

  • Scaling and Rank for numeric features:

    • Tree-based models doesn’t depend on them.
    • Non-tree-based models hugely depend on them.

  • MinMaxScaler - to [0,1]

    • X = (X X.min())/(X.max() X.min())
    • sklearn.preprocessing.MinMaxScaler
  • StandardScaler - to mean==0, std==1
    • X = (X X.mean())/X.std()
    • sklearn.preprocessing.StandardScaler
  • Rank - sets spaces between sorted values to be equal
  • np.log(1+x) and np.sqrt(1+x)

Feature generation

  • Prior knowledge
  • EDA (Exploratory Data Analysis)

Categorical features

  • Values in ordinal features are sorted in some meaningful order.
  • Label encoding maps categories to numbers.
  • Frequency encoding maps categories to their frequencies.
  • Label and Frequency encodings are often used for treebased models.
  • One-hot encoding is often used for non-tree-based models.
  • Interactions of categorical features can help linear models and KNN.

Datatime and Coordinate features

  • Datetime
    • Periodicity
    • Time since row-independent/dependent event
    • Difference between dates
  • Coordinates
    • Interesting places from train/test data or additional data
    • Centers of clusters
    • Aggregated statistics

Handling missing values

  • The choice of method to fill NaN depends on the situation
  • Usual way to deal with missing values is to replace them with -999, mean or median
  • Missing values already can be replaced with something by organizers
  • Binary feature “isnull” can be beneficial
  • In general, avoid filling nans before feature generation
  • Xgboost can handle NaN

Feature extraction from text and images

Texts

  • Preprocessing
    • Lowercase, stemming, lemmarization, stopwords
  • Bag of words
    • Huge vectors
    • Ngrams can help to use local context
    • TFiDF can be of use as postprocessing
  • Word2vec
    • Relatively small vectors
    • Pretrained models

Images

  • Features can be extracted from different layers
  • Careful choosing of pretrained network can help
  • Finetuning allows to refine pretrained models
  • Data augmentation can improve the model
